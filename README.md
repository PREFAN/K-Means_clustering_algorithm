# K-Means_clustering_algorithm
This Python script provides a comprehensive demonstration of the K-Means clustering algorithm implemented from scratch. The main objective is to illustrate how the choice of initial centroids significantly impacts the final clustering results, a well-known challenge in unsupervised machine learning.
The code begins by generating two distinct synthetic 2D datasets using NumPy. The first dataset consists of 1,000 points randomly distributed in a uniform manner across the unit square (0 to 1 on both axes), representing data with no inherent cluster structure. The second dataset contains 1,000 points organized into four clear, well-separated Gaussian clusters centered at different locations. This contrast allows for an effective comparison between unstructured and highly structured data.
A custom k_means() function is then implemented manually, without relying on scikit-learn’s built-in KMeans. The function performs the standard two-step K-Means process for a fixed number of iterations (5 in this case). In the assignment step, each data point is assigned to the nearest centroid based on squared Euclidean distance. In the update step, each centroid is recalculated as the mean of all points currently assigned to it. This process is repeated to refine the cluster assignments and centroid positions.
To highlight the importance of initialization, the script runs the K-Means algorithm multiple times with different starting centroid positions for both datasets. For the uniform dataset, one initialization places centroids near the corners, while another places them near the midpoints of the edges. For the clustered dataset, one initialization starts at the actual cluster centers, while the other uses slightly offset positions. The final centroid locations after convergence are printed for each run.
The most insightful part of the project is the visualization. Using Matplotlib, the script creates a 2x2 grid of scatter plots. Each subplot displays the data points colored according to their assigned cluster, with the final centroids marked as large red 'x' markers. These visual comparisons clearly show how poor initialization can lead to suboptimal or incorrect clustering, especially on the uniform dataset, while good initialization produces clean and accurate clusters on the structured dataset.
Overall, this code serves as an excellent educational tool. It helps users understand the inner workings of the K-Means algorithm, the sensitivity of the method to initial centroid selection, and the importance of careful initialization strategies (such as K-Means++) in real-world applications. By implementing the algorithm manually and providing clear visual feedback, the project reinforces fundamental concepts in unsupervised learning and clustering analysis.
